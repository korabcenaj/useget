# Support

## Getting Help

Use the repository issue tracker for:

- reproducible bugs
- feature requests
- documentation gaps

Before opening an issue:

- review the [README.md](README.md)
- review the [CONTRIBUTING.md](CONTRIBUTING.md)
- confirm the issue is not already reported

## Security Issues

Do not report security-sensitive issues in public tickets.

Follow the process in [SECURITY.md](SECURITY.md) for responsible disclosure.

## Pull Requests

If you already have a fix or improvement ready, open a pull request against the appropriate branch for the current workflow.
