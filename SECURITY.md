# Security Policy

## Supported Branches

Security fixes should target the currently active production branches:

- `main`
- active `release/*` branches

## Reporting A Vulnerability

Do not open public issues for security-sensitive problems.

Instead, report the issue privately to the repository owner with:

- a clear summary of the vulnerability
- reproduction steps or proof of concept
- impact assessment
- any suggested remediation

The goal is to acknowledge reports quickly, validate the issue, and ship a fix through the release workflow before public disclosure.
